# M1.9.2.1.2 — PHPUnit Bridge Runtime Hotfix

## Scopo

Correggere il secondo falso positivo emerso eseguendo il gate su una working copy già usata.

Symfony PHPUnit Bridge scarica automaticamente la propria distribuzione di PHPUnit sotto `bin/.phpunit/`. Questa directory contiene un albero completo di dipendenze Composer, incluso codice PHP e file `composer.lock`; il release manifest checker della M1.9.2.1.1 li interpretava quindi come nuovi sorgenti applicativi non presenti nel manifest.

## Correzione

- `bin/.phpunit/` è classificata esplicitamente come tooling runtime generato;
- il release manifest checker la ignora quando controlla file sorgente inattesi;
- un manifest non può comunque dichiarare file sotto `bin/.phpunit/` come file di release;
- il package audit rigoroso segnala `bin/.phpunit/` se compare nel tree usato per costruire lo ZIP;
- `.gitignore` esclude `bin/.phpunit/`;
- test dedicati riproducono l'albero reale `bin/.phpunit/phpunit/vendor/...`.

## Risultato operativo

Il gate può essere rilanciato dopo una precedente esecuzione PHPUnit senza cancellare la directory scaricata dal Symfony PHPUnit Bridge. L'integrità dei sorgenti distribuiti resta verificata tramite SHA-256 e il controllo di confezionamento resta rigoroso.

Gate:

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\scripts\verify-m1.9.2.1.2.ps1
```

```sh
./scripts/verify-m1.9.2.1.2.sh
```
