# M1.9.2.1.1 — Verification Workflow Hotfix

## Scopo

Correggere il gate M1.9.2.1 quando viene eseguito su una working copy già inizializzata.

Il precedente script lanciava `tools/package-audit.php` sul filesystem corrente. Dopo un bootstrap valido, Symfony e Composer generano intenzionalmente `.env.local`, `vendor/`, cache, database, log e altri file sotto `var/`; il package audit li segnalava quindi come se fossero stati distribuiti nello ZIP.

## Separazione dei due controlli

La verifica distingue ora due responsabilità:

1. **Package audit rigoroso** — `php tools/package-audit.php`
   - resta il controllo di confezionamento/release;
   - deve essere eseguito sul tree pulito prima della distribuzione;
   - continua a rifiutare segreti locali, `vendor/`, database, cache e altri artefatti runtime.

2. **Release manifest verification** — `php tools/release-manifest-check.php`
   - è il controllo usato dal gate installabile;
   - verifica SHA-256 di tutti i sorgenti appartenenti alla release;
   - rileva file sorgente mancanti, modificati o inattesi;
   - non considera `.env.local`, `vendor/` e `var/*` come parte dell'integrità della release, perché possono essere generati legittimamente dopo l'estrazione;
   - rifiuta comunque un manifest che tenti di dichiarare file runtime o secret-bearing come file distribuibili.

## Risultato operativo

`scripts/verify-m1.9.2.1.ps1` e `.sh` possono ora essere rieseguiti in modo idempotente anche dopo bootstrap e test precedenti, senza richiedere la cancellazione manuale della working copy.

Alias espliciti della hotfix:

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\scripts\verify-m1.9.2.1.1.ps1
```

```sh
./scripts/verify-m1.9.2.1.1.sh
```

Il package audit rigoroso non è stato indebolito: è stato semplicemente spostato nel corretto confine di responsabilità, cioè il processo di costruzione della release.

## Estensione M1.9.2.1.2

Una successiva esecuzione ha mostrato un'altra categoria di file generati: `bin/.phpunit/`, popolata automaticamente dal Symfony PHPUnit Bridge con PHPUnit e le sue dipendenze. La M1.9.2.1.2 estende la stessa separazione di responsabilità anche a questa directory.
