# M1.9.4.1 — Accounting Schema Enforcement Hotfix

## Origine

La prima validazione utente di M1.9.4 ha prodotto un risultato anomalo ma molto utile:

- suite PHPUnit completa: verde;
- gate `app:verification:play-start-accounting --env=test`: rosso solo su `Duplicate accounting protection`;
- i tre inserimenti duplicati `PLAYER_ENTRY`, `JACKPOT_CONTRIBUTION` e `ORGANIZER_SHARE` risultavano accettati.

Questo dimostra una divergenza tra lo schema SQLite atteso dal codice M1.9.4 e lo schema effettivamente usato dal gate. Il gate originario provava il comportamento, ma non diagnosticava prima la presenza degli oggetti DB richiesti.

## Correzioni

### Migration `Version20260719000400`

La migration ricrea esplicitamente:

- `uniq_standard_player_entry_per_play`;
- `uniq_standard_jackpot_contribution_per_play`;
- `uniq_standard_organizer_share_per_play`;
- `trg_ledger_standard_entry_play_binding`.

Il trigger viene inoltre rafforzato: una seconda riga con lo stesso `(play_id, entry_type)` viene respinta esplicitamente, indipendentemente dal `correlation_id` proposto.

Gli indici unici restano una seconda barriera strutturale.

### Gate auto-diagnostico

Prima dello scenario funzionale, `PlayStartAccountingGateVerifier` interroga `sqlite_master` e richiede:

- tre indici unici;
- un trigger di binding.

Il report deve mostrare:

```text
Accounting schema hardening: 3 unique indexes + 1 binding trigger present
```

### Sincronizzazione immediata del database test

`verify-m1.9.4.ps1/.sh` esegue ora:

```bash
php bin/console doctrine:migrations:migrate --no-interaction --env=test
```

immediatamente prima del gate M1.9.4.

In questo modo il gate non dipende dallo stato residuo lasciato da precedenti esecuzioni PHPUnit o da una working copy già inizializzata.

## Gate corrente

Windows:

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\scripts\verify-m1.9.4.1.ps1
```

Unix:

```bash
./scripts/verify-m1.9.4.1.sh
```

Il gate è valido solo se `Duplicate accounting protection` restituisce:

```text
entry=blocked, jackpot=blocked, organizer=blocked
```

oltre a tutti gli altri controlli verdi.
